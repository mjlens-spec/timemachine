# 证据分级 · S / A1 / A2 / B / C

> 照搬现有「AI 营销方案工作流 SOP」的证据等级体系，**不另造**。全流程的采集、清洗、引用、洞察锚定都以此为准。

| 等级 | 来源 | 使用原则 |
|---|---|---|
| **S 级** | 客户一手数据（历史投放、CRM、销售、内部 BI） | 最高优先级，核心论据首选；需确认客户授权引用范围。**原始件放 `_raw/`，不入库。** |
| **A1 级** | 平台官方后台（巨量云图、灵犀、聚光、蒲公英、星图） | 可直接作核心结论，优先引用 |
| **A2 级** | 高可信第三方（飞瓜） | 直播/带货场景可作核心论据，需标注"数据来源为第三方估算" |
| **B 级** | 第三方分析平台（蝉妈妈、千瓜） | 辅助验证 A 级结论，交叉印证后可引用 |
| **C 级** | 公开信息（Perplexity / Gemini / Google、行业报告、媒体） | 仅作背景铺垫，**不单独支撑核心结论** |

## 在时间机器里的两条硬规矩

1. **每条断言都带级别**。研究综述 `03-research-synthesis.md` 和洞察画布 `04-insight-canvas.md` 里，凡是事实/数据断言，后面都跟一个 `（来源·级别）`。
2. **洞察锚点的核心断言需 ≥ B 级支撑**。
3. **测算的每个数字也要标出处**。`06-calculation.md` 里每个数要么带 `（来源·级别）`，要么带 `【假设：___】`；纯 C 级 / 全假设的测算标「示意」，不作承诺值。
   - 只有 C 级支撑的核心结论 → 一律标 **「待验证」**，不可作为锚点洞察。
   - 这是反直觉标尺第 6 条（见 `counter-intuition-bar.md`）的判据。

## Claude 能拿到 / 拿不到什么

- **能自动拿（→ 多为 C 级）**：WebSearch / WebFetch 公开信息；行业报告、媒体稿、竞品公开动作。
- **能从 Notion 拿（级别随原始来源）**：项目知识库里的过往案例、研究、客户提供过的数据。
- **拿不到（需你粘贴）**：A1/A2/B 的平台后台数据（巨量云图/灵犀/聚光/蒲公英/星图/飞瓜/蝉妈妈/千瓜）——Claude 登不了这些后台。时间机器会**明确请你粘贴指定切片**，并在入库时**当场打级别**、原始件落 `_raw/`。粘贴请求模板见 `research-playbook.md`。
