# 洞察引擎 · 操作手册

> 阶段 3 的完整玩法。SKILL.md 里只放骨架；深入看这里。
> 核心信念：**洞察不是顺数据滑出来的，是被"逼"出来的。** 研究回答"什么是真的"，时间机器回答"什么是真的、但别人还没据此行动"。
> 交互范式对齐 `/mnt/skills/examples/doc-coauthoring/SKILL.md`：发散多候选 → 人 keep/kill/combine → 起草 → 自我反驳 → 迭代。AI 负责"产出 + 反驳"，人负责真实判断、客户认知、"哪条让我意外、想停一下"的方向感。

## 四个洞察切入点（逐个来，不要混着上）

输入素材来自 `03-research-synthesis.md` 的 **§10 矛盾与异常清单** 和 `00-positioning.md` / `01-brief-decode.md` 的 **品类常识清单与洞察先行**。
顺序：**张力 → 假设反转 → JTBD → 文化位移**。每个切入点给 3–5 条编号候选。

### ① 张力 / 矛盾捕捉
- **干什么**：找"人嘴上说的 ≠ 实际做的"，或"品类惯例 ≠ 真实欲望"的切口。
- **什么时候咬得动**：评论区/搜索词和品牌官方话术对不上；A1 人群数据和品牌自我认知打架。
- **模板**：`用户嘴上说 X（来源·级别），但行为/数据显示他们做 Y（来源·级别）。张力 = ___。`
- **✅ 反直觉范例**：保温壶品类都在卷"保温时长/参数"，但高互动评论里用户晒的全是"露营、雪山、加班路上那口热水的踏实"。张力＝**卖的是参数，买的是"在外面也有家的温度"**。→ 重构为户外情绪而非 24 小时保温。
- **❌ 顺的陷阱**："用户既要质量好又要价格低"——人人皆知，不是张力，是废话。

### ② 假设反转
- **干什么**：把『品类常识清单』逐条反过来，用证据验真，看反面是否站得住、能否长出策略。
- **模板**：`常识：___；反转：___；支持反转的证据：___（级别）；反转站得住吗？能长出动作吗？`
- **✅ 反直觉范例**：常识"啤酒要打爽/痛快/兄弟局，人群越泛越好"。反转"**不追泛人群，先把'轻卡低负担'这一小群做深，由泛到精占品类心智**"。证据：轻卡赛道心智规模小但增速与复购更高（B）。站得住。
- **❌ 顺的陷阱**：为反而反——"啤酒不该好喝"。反面无证据、不可转化，丢弃。
- **提醒**：不是每条常识反转都成立。只留"反转后有 ≥B 级证据 + 能长出差异化动作"的。

### ③ 真实动机 / JTBD
- **干什么**：用户到底在"雇"这个产品完成什么**情感/社交任务**，而不是它的功能。
- **什么时候咬得动**：评论区的"使用场景/购买动机/犹豫点/晒单理由"最藏 JTBD。
- **模板**：`用户其实是雇这个产品来完成 ___（情感/社交任务），而非它的功能。证据：评论区/搜索词 ___（级别）。`
- **✅ 反直觉范例**：儿童营养品——妈妈买的不是"补钙功能"，是"**我是个尽责好妈妈**"的自我确认，以及在妈妈群里"不能输"的社交安全感。功能只是入场券，社交身份才是扳机。
- **❌ 顺的陷阱**："用户想要健康/想变美"——太普世，所有人所有品类都成立。

### ④ 文化 / 情境位移
- **干什么**：世界上正在变的什么，让那条旧"真理"失效、或打开一道新缝？
- **模板**：`过去成立的真理是 ___；正在变的是 ___（趋势·级别）；这让旧真理失效 / 打开新机会 ___。`
- **✅ 反直觉范例**：进口啤酒过去＝"酒吧夜店的小众高端调性（小而美）"。变化＝年轻人把"微醺/松弛/生活场景"搬进日常与社媒。于是从"小而美"**位移到"广而效"**——锚定生活场景而非夜店，实现破圈。
- **❌ 顺的陷阱**："现在是 AI 时代 / 大家都刷短视频"——太大太泛，不指向这个品类的具体机会。

## 共创回合 A → D

### Round A · 发散（逐个）
对每个切入点，从输入素材里提 **3–5 条编号候选**，每条带证据来源·级别。一个切入点讲完，停下来问人：

> **"这几条里哪条让你『意外、想停一下』？哪条是废话？为什么？"**

人用速记回（"留 2、4；杀 1（太顺）；并 3+5"）。把这个走向**学下来带到下一个切入点**——人的取舍在暴露他的判断标准，越往后越准。

### Round B · 收敛
跨四个洞察切入点，把存活的碎片综合成 **2–3 个成形候选**。每个写成：
- 一句话洞察（可被一字不差复述）
- 两行"为何成立 / 它解释了什么矛盾"

### Round C · 反驳（对自己的洞察下手）
对**每个**成形候选，执行"一个产出·一个反驳·一个定稿"——AI 分别扮演三个**具名反方**，必须给出**具体**反制，不许说含糊的"可能有风险"：

- 🥊 **竞品策略负责人**："如果我是竞品，我会这样反制 / 这样跟进，让你的洞察失效——___。"
- 🧑‍💼 **客户品牌负责人**："这条我不信，因为 ___；我会要你拿出 ___ 证据。"
- 🔬 **数据审查官**："这条只有 C 级支撑 / 与 ___ 数据矛盾 → 标『待验证』。"

每条反驳后，AI **诚实**判定：洞察是**存活**、**需重构**、还是**阵亡**。

> ⚙️ **可选升级**：若要更狠、避免"自己夸自己"的确认偏差，把反方派给一个子 agent（`Agent(subagent_type="general-purpose")`），只喂洞察画布草稿 + 反方人设，让它独立挑刺。默认单 agent 角色扮演即可。

### Round D · 定稿
人选/并/改 → AI 重写赢家 → 过 `counter-intuition-bar.md` 六点标尺。
**不过标尺，就退回 Round A 在最弱那一维重逼，绝不为了通过而降标尺。**

## 操作要点

- **"意外、想停一下"是信号**：人突然停顿、皱眉、"咦"——那条候选大概率有反直觉的料，追下去。
- **一个切入点只攻一个切口**：别一段话里四个切入点一起上，人跟不上、也评不准。
- **碎片可以拼合**：常常是"张力"提供了切口、"文化位移"解释了为什么现在裂、"JTBD"说明切口下面是什么——拼合起来才是完整洞察。
- **宁可承认没有**：如果异常清单本就单薄、逼不出真反直觉，**老实说**，给出最强的"顺洞察"并明确标注"这是顺的"，建议补研究——不要硬编一个假的反直觉。
- **锚点只要一个**：原则上一个项目落一个锚点洞察；若两条都过了标尺且不打架，可作"双锚"，但要在画布 §0 说明主次。
- **候选清单别删**：没采用的候选连同淘汰原因留在画布 §7——客户万一毙了主洞察，runner-up 能救场。
