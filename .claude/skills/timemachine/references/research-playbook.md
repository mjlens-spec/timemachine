# 自动洞察研究 · Playbook

> 阶段 3 怎么跑。**一句话原则：研究只为四把刀服务，以矛盾/异常为先——不是再做一遍全量采集。**

## 先分模式

```
已经有 Research Brief（Notion 链接 / 你粘贴）？
├─ 有 → INGEST 模式：吃掉它，映射到四把刀，只对阶段 1–2 的矛盾点做"定向补搜"。不要重采。
└─ 无 → LEAN 模式：跑一遍精研（下表），目标是凑齐四把刀的弹药 + 填满 §10 异常清单。
```

> ⚠️ 时间机器**不替代**现有 SOP 的 STEP 2–6 全量采集。它要么吃现成 Research Brief，要么只做轻量的洞察导向精研。

## 多 Subagent fan-out（LEAN 模式的核心动作）

全网研究不要单线程一条条搜。派**多个子 agent**（`Agent`，subagent_type 用 `Explore` 或 `general-purpose`）各包一路，并行跑，结果汇进 `03-research-synthesis.md` 的 §0 综述摘要：

- 子 agent 1 · **品类趋势**：行业规模/增速、细分赛道、正在涨的需求。
- 子 agent 2 · **竞品公开动作**：竞品近期内容/投放/产品/达人的公开信号。
- 子 agent 3 · **文化/情境位移**：什么在变、让旧真理失效或打开新缝。
- 子 agent 4 · **人群讨论**：评论区/搜索词/社媒里的真实说法与犹豫点。

每个子 agent 只喂它那一路的任务 + 客户/品类/空位上下文，让它独立产出，回来你再缝。各路都是 C 级公开信息（背景用），真正的 A1/A2/B 来自数据同事回传或用户粘贴。

> 数据同事回传从哪来：阶段 2b 产出的 `02-data-marketing-brief.md`（云图/灵犀/星图取数单）。回传的 A1 数据吃进 §3/§4，平台后台拿不到的再用下面的粘贴模板兜底。

## 向你要平台数据的粘贴请求模板（按需发）

> 每段粘贴进来，AI **当场打证据级别**、原始件/截图落 `_raw/`、工件里只留脱敏结论。

- **巨量云图（A1）**："请粘贴巨量云图近 90 天：本品 vs 竞品的人群画像（年龄/性别/地域/兴趣 TGI）、5A 人群规模差、行业内容热点。"
- **灵犀（A1）**："请粘贴灵犀近 30 天：目标人群在小红书的兴趣/内容偏好、目标品类趋势与细分赛道机会、本品 vs 竞品的人群/心智差异。"
- **聚光（A1）**："请粘贴聚光：目标行业投放规模与节奏、竞品广告形式偏好（信息流/搜索/直播）、人群种草路径。"
- **蒲公英（A1）/ 星图（A1）**："请粘贴目标品类活跃达人（互动率>X%）、竞品近期合作达人组合、已合作达人效果复盘。"
- **飞瓜（A2）**："请粘贴竞品近期直播 GMV/转化率趋势、高转化达人组合、商品定价与动销节奏。"
- **蝉妈妈（B）/ 千瓜（B）**："请粘贴竞品内容种草链路、品类达人分层矩阵、近期爆款笔记的标题/封面/话题规律。"

## 把研究汇成 §10 异常清单

研究边跑边记**对不上的地方**：说 vs 做、品类惯例 vs 真实欲望、过去成立 vs 最近变了。这些进 `03-research-synthesis.md` 的 §10，是阶段 4 洞察引擎的发射台。研究做完时，§8 风险与反证、§10 异常清单**都必须非空**——空了说明还没挖到能逼洞察的料。
